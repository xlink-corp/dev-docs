## 1. 模块概述

### 1.1 功能定位

巡检管理模块是设备运维管理的核心组件，用于规范设备巡检流程、确保设备正常运行、预防故障发生。

### 1.2 核心价值

| 价值维度 | 说明 |
|---------|------|
| 标准化执行 | 通过模板统一巡检内容和标准 |
| 流程可追溯 | 完整记录巡检过程和发现问题 |
| 问题闭环 | 问题发现 → 工单生成 → 处理 → 验收的完整流程 |
| 数据分析 | 多维度统计分析，支持决策优化 |

### 1.3 用户角色与权限

| 角色 | 主要职责 | 核心权限 |
|------|---------|---------|
| 系统管理员 | 模块配置、权限管理 | 全部权限 |
| 设备管理员 | 模板管理、计划制定、任务监控 | 模板创建/编辑、计划管理、报告查看 |
| 班组长 | 任务分配、执行监督、问题审核 | 任务分配、执行记录查看、问题审核 |
| 巡检员 | 任务执行、问题上报 | 任务执行、问题上报、个人记录查看 |

### 1.4 业务流程总览

```
模板配置 → 计划制定 → 任务生成 → 任务执行 → 问题处理 → 统计分析
    ↑                                              |
    └──────────── 持续优化 ←────────────────────────┘
```

## 2. 快速入门

### 2.1 首次配置（设备管理员）

#### 步骤 1：创建巡检模板

1. 进入 **巡检管理 → 巡检模板**
2. 点击 **新建模板**
3. 填写基本信息：
   - 模板名称（如：空调主机日常巡检）
   - 模板编码（系统自动生成或自定义）
   - 适用设备类型
4. 添加巡检项目：
   - 点击 **添加检查项**
   - 选择检查类型：布尔值（是/否）、数值（带上下限）、枚举（选项）、文本、拍照
   - 设置是否必填、是否需要拍照
5. 保存并发布模板

#### 步骤 2：创建巡检计划

1. 进入 **巡检管理 → 巡检计划**
2. 点击 **新建计划**
3. 配置计划信息：
   - 计划名称
   - 关联模板
   - 巡检范围（选择设备或区域）
   - 执行周期（每日/每周/每月/自定义）
   - 执行时间窗口
   - 指定执行人或班组
4. 启用计划

#### 步骤 3：验证任务生成

1. 进入 **巡检管理 → 任务执行**
2. 确认系统已按计划生成巡检任务
3. 通知执行人员开始巡检

### 2.2 日常巡检执行（巡检员）

#### 使用移动端执行巡检

1. **登录系统**：打开移动端应用或 H5 页面
2. **查看待办任务**：进入"我的任务"查看分配的巡检任务
3. **开始巡检**：
   - 点击任务进入执行页面
   - 扫描设备二维码或选择设备
   - 逐项填写检查结果
4. **问题上报**：
   - 发现异常时点击"上报问题"
   - 选择问题类型、严重程度
   - 添加照片和描述
   - 选择是否需要生成工单
5. **提交任务**：完成所有检查项后提交

## 3. 功能模块详解

### 3.1 巡检模板管理
<img width="1712" height="938" alt="QQ_1769754227055" src="https://github.com/user-attachments/assets/a33aab91-cab8-43cb-991c-1476d1d18fe9" />


#### 3.1.1 模板列表

**访问路径**：巡检管理 → 巡检模板

**页面功能**：
- 查看所有模板列表
- 按状态、类型、关键词筛选
- 创建、编辑、复制、删除模板
- 查看模板使用情况

**字段说明**：

| 字段 | 说明 |
|------|------|
| 模板名称 | 巡检模板的显示名称 |
| 模板编码 | 唯一标识码 |
| 版本 | 模板版本号，编辑后自动升级 |
| 状态 | 草稿/已发布/已停用 |
| 检查项数量 | 包含的巡检项目数量 |
| 创建时间 | 模板创建时间 |
| 使用次数 | 被计划引用的次数 |

#### 3.1.2 创建/编辑模板

**基本信息**：
- 模板名称（必填）
- 模板编码（必填，唯一）
- 模板描述
- 适用设备类型
- 预计执行时长

**检查项配置**：

每个检查项包含以下属性：

| 属性 | 说明 | 示例 |
|------|------|------|
| 项目名称 | 检查项显示名称 | 运行温度 |
| 检查类型 | boolean/number/enum/text/photo | number |
| 是否必填 | 执行时是否必须填写 | 是 |
| 标准值范围 | 数值类型的合格范围 | 35-45°C |
| 选项列表 | 枚举类型的可选值 | 正常/异常/待观察 |
| 需要拍照 | 是否强制拍照 | 否 |
| 备注说明 | 执行时的提示信息 | 使用红外测温枪测量 |

#### 3.1.3 模板版本管理

- 发布后的模板修改会创建新版本
- 进行中的计划继续使用原版本
- 新计划自动使用最新版本
- 可查看历史版本内容对比

### 3.2 巡检计划管理
<img width="1710" height="906" alt="QQ_1769754943903" src="https://github.com/user-attachments/assets/48f1312d-d949-467f-8731-11cf76818b39" />


#### 3.2.1 计划列表

**访问路径**：巡检管理 → 巡检计划

**页面功能**：
- 查看所有巡检计划
- 按状态、周期类型筛选
- 创建、编辑、启用/停用计划
- 查看计划执行统计

#### 3.2.2 创建/编辑计划

**基本信息**：
- 计划名称（必填）
- 计划编码
- 关联模板（必填）
- 计划描述

**巡检范围**：
- 按设备选择：选择具体设备
- 按区域选择：选择空间/位置，包含其下所有设备
- 按设备类型：选择某类型的所有设备

**执行周期**：

| 周期类型 | 配置项 |
|---------|-------|
| 每日 | 执行时间 |
| 每周 | 星期几 + 执行时间 |
| 每月 | 日期 + 执行时间 |
| 自定义 | 间隔天数 + 起始日期 |

**执行人配置**：
- 指定执行人：固定人员执行
- 指定班组：班组内轮换或分配
- 不指定：任务生成后手动分配

**高级设置**：
- 任务提前生成时间
- 超时自动关闭
- 漏检提醒
- 异常自动生成工单

#### 3.3.2 执行巡检

**PC端执行**：
1. 在任务列表点击"执行"
2. 进入执行详情页
3. 逐项填写检查结果
4. 异常项标记并添加说明
5. 上传相关照片
6. 提交完成

**移动端执行**：
1. 移动端登录后查看"我的任务"
2. 点击待执行任务
3. 可扫码定位设备
4. 填写检查结果
5. 实时拍照上传
6. 发现问题可直接上报
7. 提交完成

- 查看任务详情和填写内容
- 查看检查项明细和照片
- 查看异常项和问题记录
- 导出执行报告

#### 3.6.1 统计分析

**访问路径**：巡检管理 → 统计分析

**分析维度**：
- 时间维度：日/周/月/年
- 组织维度：按班组/部门

| 平均执行时长 | 任务平均耗时 | 总耗时 / 任务数 |
| 漏检率 | 超时未执行的占比 | 超时数 / 计划数 × 100% |

## 4. 常见问题 FAQ

### 4.1 模板相关

**Q: 已发布的模板可以修改吗？**

A: 可以。修改后会生成新版本，进行中的计划继续使用原版本，新计划使用新版本。

**Q: 如何复用其他模板的检查项？**

A: 在模板编辑页面，可以选择"从其他模板导入"，选择要复制的检查项。

**Q: 检查项的顺序可以调整吗？**

A: 可以。在模板编辑页面，拖拽检查项可以调整顺序。

### 4.2 计划相关

**Q: 计划停用后已生成的任务会怎样？**

A: 已生成的任务不受影响，可以继续执行。停用只影响后续任务生成。

**Q: 临时需要增加一次巡检怎么操作？**

A: 可以使用"临时任务"功能，手动创建一次性巡检任务，不影响原有计划。

**Q: 节假日如何跳过巡检？**

A: 在计划的高级设置中，可以配置排除日期或启用节假日自动跳过。

### 4.3 执行相关

**Q: 巡检过程中手机没信号怎么办？**

A: 移动端支持离线执行，数据会保存在本地，恢复网络后自动同步。

**Q: 发现问题必须生成工单吗？**

A: 不是必须的。轻微问题可以只记录，严重问题建议生成工单跟进处理。

**Q: 可以补录之前的巡检记录吗？**

A: 可以，但会标记为"补录"，并记录实际操作时间。

### 4.4 权限相关

**Q: 普通巡检员可以看到其他人的任务吗？**

A: 默认只能看到自己的任务。班组长可以看到本班组的任务。

**Q: 如何设置某人只能执行特定区域的巡检？**

A: 在用户权限设置中，可以配置数据权限，限制可见的空间范围。
