## 1. 模块概述

### 1.1 功能定位

“组织碳排放”模块用于企业温室气体（GHG）盘查与管理，覆盖：

- **数据准备**：排放源配置、排放因子库、活动数据（Activity Data）
- **核算管理**：核算批次、Scope 1/2/3 汇总、状态流转与审阅
- **分析与改进**：趋势、分项、区域、碳强度、减排潜力
- **目标与输出**：减排目标跟踪、碳报告与导出

### 1.2 菜单页总览（系统路径）

本手册覆盖以下全部菜单页（与系统侧边栏一致）：

1. `/carbon-dashboard` 碳排放概览
2. `/carbon-accounting` 排放核算
3. `/activity-data` 活动数据
4. `/carbon-factors` 排放因子库
5. `/carbon-sources` 排放源配置
6. `/material-types` 物质类型
7. `/gas-types` 气体类型
8. `/organizational-boundary` 组织边界
9. `/carbon-analysis` 碳排放分析
10. `/carbon-reports` 碳报告
11. `/carbon-targets` 减排目标
12. `/suppliers` 供应商管理

### 1.3 核心概念速览

| 概念 | 说明 |
|------|------|
| Scope 1 | 直接排放（如固定燃烧、移动燃烧、制冷剂泄漏等） |
| Scope 2 | 间接排放（主要为外购电力/蒸汽/热力/冷量等） |
| Scope 3 | 其他间接排放（供应链、运输、废弃物、商务出行等） |
| 活动数据（Activity Data） | “发生了多少”的业务数据（用量/里程/产量/采购量等），是核算输入 |
| 排放因子（EF） | 将活动数据换算为排放量的系数（单位需匹配） |
| tCO2e | 二氧化碳当量；不同温室气体按 GWP 折算为 CO2e |
| 核算批次 | 将某一周期（如月/季/年）的核算数据组织成一次“版本”并进行审阅 |

### 1.4 用户角色与权限（建议）

系统中该模块统一使用 `carbon:view` 等权限（以组织权限配置为准）。推荐按岗位进行职责划分：

| 角色 | 主要职责 | 常用页面 |
|------|----------|----------|
| 碳管理员 / ESG 专员 | 主数据维护、核算执行、导出报告 | 活动数据、因子库、排放源、核算、报告 |
| 能源管理员 | Scope 2 相关数据口径、能耗侧支撑 | 活动数据、分析、概览 |
| 审阅/管理层 | 审核核算结果与目标达成 | 概览、核算、分析、目标、报告 |

## 2. 快速入门（场景化）

### 场景 A：从 0 开始完成一次核算（推荐路径）

1. **组织边界**（/organizational-boundary）：确认盘查组织范围、纳入的设施/组织单元、权益比例口径
2. **排放源配置**（/carbon-sources）：建立“排放从哪里来”的清单，并标注 Scope 与分类
3. **排放因子库**（/carbon-factors）：为不同活动类型维护正确的排放因子（注意单位一致）
4. **活动数据**（/activity-data）：按周期录入用量/里程/采购量等活动数据，并关联排放源/供应商/空间等维度
5. **排放核算**（/carbon-accounting）：创建核算批次，汇总本期 Scope 1/2/3 排放量，提交审阅
6. **碳排放分析**（/carbon-analysis）：核对趋势与热点，识别异常与减排机会
7. **碳报告**（/carbon-reports）：导出/生成对外披露或内部汇报报告

### 场景 B：快速定位“排放热点”并形成行动清单

1. 进入 **碳排放分析**（/carbon-analysis）
2. 选择时间范围（建议近 30 天或本核算期）
3. 优先查看：
   - **分项排放**：找出贡献最高的排放源/能源类型
   - **区域分析**：锁定高排放区域/空间
   - **减排潜力**：查看系统建议的机会清单与优先级
4. 对热点项回到 **活动数据/排放源** 页面核对：单位、因子、关联关系是否正确

### 场景 C：建立减排目标并持续跟踪

1. 进入 **减排目标**（/carbon-targets）
2. 新建目标：确定基准年、基准值、目标年与目标值（或减少比例）
3. 目标启用后：
   - 在概览/分析中关注目标进度
   - 每个核算周期更新活动数据与批次，确保趋势可比

## 3. 功能模块详解（按菜单页）

### 3.1 碳排放概览（/carbon-dashboard）

**页面用途**：用仪表板视角查看当前组织的碳排放 KPI，包括总排放、Scope 构成、趋势与主要来源。

**常见操作**：
- 切换核算周期/批次（如系统支持）
- 快速查看 Scope 1/2/3 构成与占比
- 从“Top 排放源/趋势异常”跳转到明细页面核对

**建议关注指标**：
- 总排放（tCO2e）
- Scope 1/2/3 分项
- 主要排放源 Top N
- 与目标的差距/进度（若已配置目标）

---

### 3.2 排放核算（/carbon-accounting）

**页面用途**：管理“核算批次”，以批次为单位汇总排放并进行审阅/留痕。

**关键字段**（页面列表常见列）：

| 字段 | 说明 |
|------|------|
| 批次名称 | 例如“2026-01 月度核算” |
| 核算期间 | period_start 至 period_end |
| 总排放量 | 当前批次总 tCO2e |
| Scope 1/2/3 | 各范围排放量 |
| 状态 | 草稿/审核中/已批准/已驳回 |
| 创建时间 | 批次创建时间 |

**状态说明**：

| 状态 | 含义 | 建议动作 |
|------|------|----------|
| 草稿（draft） | 仍在补数/核对 | 完善活动数据与因子，准备提交 |
| 审核中（under_review） | 进入审阅流程 | 仅修正必要问题，避免反复变更 |
| 已批准（approved） | 结果可用于报告与对外披露 | 锁定口径、用于对比与目标跟踪 |
| 已驳回（rejected） | 需修正后再提审 | 根据驳回意见回溯活动数据/因子 |

---

### 3.3 活动数据（/activity-data）

**页面用途**：录入/管理核算输入数据（用量、里程、采购量等），并支持数据状态流转与质量标注。

**常见字段（示例）**：

| 字段 | 说明 |
|------|------|
| 记录日期（record_date） | 数据发生日期 |
| 数据来源（source_type） | 数据来自哪里（如表计、发票、人工等） |
| 活动类型（activity_type） | 用于标识数据类别（如用电量、燃料用量等） |
| 活动数据值/单位 | 核算输入值及单位（必须与因子单位匹配） |
| 设施/空间（space） | 可选，用于区域归因 |
| 供应商（supplier） | 可选，常用于 Scope 3 |
| 状态（status） | 草稿/已验证/已处理（用于数据治理） |
| 数据质量（data_quality） | 高/中/低/估算（建议配合审计） |

**数据状态建议**：
- 草稿：允许频繁修改
- 已验证：完成复核与必要佐证
- 已处理：已进入核算计算/批次汇总

---

### 3.4 排放因子库（/carbon-factors）

**页面用途**：维护排放因子（EF），用于将活动数据换算为排放量。

**使用要点**：
1. **年份/地区/口径**：同一活动类型在不同年份或地区可能有不同因子
2. **单位一致性**：活动数据单位必须与因子单位可换算/一致（如 kWh、L、m³、t、km 等）
3. **Scope 与类型匹配**：因子类型需与排放源/活动类型一致

---

### 3.5 排放源配置（/carbon-sources）

**页面用途**：建立“排放源台账”，用于对活动数据进行归因（Scope/分类/设施/供应商等）。

**推荐做法**：
- 使用清晰、可审计的命名：`场地-系统-设备/过程-介质`
- 为每个排放源明确 Scope（1/2/3）与类别
- 对 Scope 3 尽量绑定供应商，便于后续汇总与供应链披露

---

### 3.6 物质类型（/material-types）

**页面用途**：维护物质/材料类型主数据（用于部分 Scope 1/3 的活动数据与因子关联）。

**适用场景**：
- 原材料/辅料/包装/废弃物等与排放相关的物料分类
- 需要进行“材料维度”的排放核算或分析

---

### 3.7 气体类型（/gas-types）

**页面用途**：维护温室气体类型与 GWP（AR4/AR5/AR6 等版本），常用于制冷剂等逸散排放。

**使用提示**：
- 若系统支持选择 GWP 版本，请在组织层面统一口径
- 变更 GWP 版本会影响历史可比性，建议在核算期锁定

---

### 3.8 组织边界（/organizational-boundary）

**页面用途**：定义盘查边界与纳入的组织单元/设施，确保“统计范围一致”。

**建议内容**：
- 边界口径（运营控制法/财务控制法/权益法等，视企业规则）
- 纳入设施清单与权益比例（如有）
- 生效年份/版本说明（便于审计与回溯）

---

### 3.9 碳排放分析（/carbon-analysis）

该页面提供多 Tab 多维分析：

- **排放趋势**：按时间观察排放变化（支持导出）
- **分项排放**：按 Scope / 排放源 / 能源类型拆解结构
- **区域分析**：按空间/区域汇总对比
- **碳强度**：单位能耗/单位产出等强度视角（如已配置相关分母）
- **减排潜力**：识别热点与建议机会清单

**使用建议**：
1. 先看趋势判断是否存在异常峰值/谷值
2. 再用分项与区域定位“贡献最大的 20%”
3. 最后结合强度与减排潜力形成行动列表

---

### 3.10 碳报告（/carbon-reports）

**页面用途**：输出标准化碳报告，用于内部汇报或对外披露（具体功能以页面实际为准）。

**建议在报告前核对**：
- 本期核算批次状态为“已批准”
- 活动数据状态已完成验证
- 因子口径与年份选择正确

---

### 3.11 减排目标（/carbon-targets）

**页面用途**：创建与跟踪减排目标，支持查看进度、编辑与删除。

**典型字段**：

| 字段 | 说明 |
|------|------|
| 目标名称 | 如“2026 年度碳排放强度下降 10%” |
| 目标类型 | 绝对量 / 强度（以页面选项为准） |
| 目标范围 | Scope 1/2/3（可单选或组合） |
| 基准年/基准值 | 作为对比起点 |
| 目标年/目标值 | 目标值或目标强度 |
| 状态 | 进行中/已完成/已暂停（以页面为准） |

**操作建议**：
- 先确定核算口径与基准年的数据完整性
- 强度目标需先明确“分母”（如产量、面积、营收等）与单位

---

### 3.12 供应商管理（/suppliers）

**页面用途**：维护供应商主数据，支撑 Scope 3 维度归因与后续供应链披露。

**建议维护的信息**：
- 供应商类型（原材料/物流/服务等）
- 碳披露状态（已披露/部分披露/未披露）
- 排放因子来源类型（供应商特定/行业平均/IPCC 等）

## 4. 数据导出与报表

### 4.1 通用导出说明

系统中多处分析页/列表支持导出（Excel/CSV 取决于页面功能），一般遵循：

- 导出内容 = **当前筛选条件**下的数据
- 文件名通常包含导出时间戳（例如：`碳排放趋势分析_YYYYMMDD_HHmmss.xlsx`）

### 4.2 “碳排放分析”常见导出

| 分析类型 | 常见导出内容 |
|---------|--------------|
| 趋势分析 | 日期、Scope1/2/3、总计 + 摘要统计（峰值/谷值/同环比） |
| 区域分析 | 区域/空间排放汇总、占比、明细 |
| 碳强度 | 能源/产品强度明细与汇总 |

## 5. 常见问题 FAQ

### 5.1 为什么我看不到任何碳数据？

可能原因：
1. 未选择组织（Organization）或当前组织无数据
2. 当前筛选时间范围内没有活动数据/排放记录
3. 权限不足（缺少 `carbon:view` 或数据范围权限）

### 5.2 为什么排放量为 0 或明显不合理？

排查顺序建议：
1. 活动数据单位是否正确（kWh/L/m³/t/km 等）
2. 排放因子单位是否与活动数据匹配
3. 排放源的 Scope 归类是否正确
4. 是否将数据错误关联到其他组织/空间/供应商

### 5.3 批次“审核中/已批准”后还能修改吗？

建议做法：
- 已批准批次用于报告与目标对比，尽量避免回改。
- 如必须修正：建议创建新的核算批次（或按系统规则退回/重审），保留历史可追溯性。
